Lokale Dienste

Für echte LAN-IPv4-HTTP-Frames diese Datei lokal öffnen oder im LAN per HTTP hosten. Beispiele: file:///..., VS-Code Live Server oder python -m http.server 8000. Dann können http://192.168.x.x-Frames geladen werden.

Kurztutorial

Login-Websites

Wähle als Typ Login-Website. Dann wird der Frame ohne Sandbox geladen. Formulare, Cookies und Login-Popups haben dadurch die besten Chancen.

Grenze: Wenn die Zielseite selbst X-Frame-Options, frame-ancestors, blockierte Third-Party-Cookies oder eine Weiterleitung aus dem Frame nutzt, kann die Regie Wall das nicht erzwingen. Nutze dann Login/Öffnen, melde dich im normalen Tab an und lade danach den Frame neu.

IPv4-/LAN-Dienste

Lokale HTTP-Geräte wie http://192.168.0.50:8080 können nur dann im Frame erscheinen, wenn die Regie Wall nicht selbst aus einem HTTPS-Kontext geladen wurde. Browser blockieren sonst Mixed Content; das kann JavaScript nicht abschalten.

  • Empfohlen: Diese HTML-Datei direkt lokal öffnen (file://) oder im LAN per HTTP hosten.
  • Beispiel im Ordner der Datei: python -m http.server 8000, danach http://localhost:8000/index(1).html öffnen.
  • Alternativ: dem lokalen Gerät HTTPS geben oder einen Reverse Proxy verwenden.

Tonsteuerung

Mute/Solo/Volume funktionieren bei YouTube und HLS. Bei fremden Website-iFrames ist Tonsteuerung aus Sicherheitsgründen nicht zuverlässig möglich.
