Installer Windows & Linux

Apps als Installer verpacken

CLI-Installer, Tkinter-Oberfläche, Startmenü-Eintrag, Desktop-Verknüpfung, Icon und Paketierung für Java- und Python-Projekte.

Ein Programm ist für dich als Entwickler oft schon fertig, wenn es lokal mit python main.py, java -jar app.jar oder einer gebauten .exe startet. Für normale Nutzerinnen und Nutzer ist es aber erst dann eine richtige App, wenn sie installiert werden kann, im Startmenü oder App-Menü auftaucht, ein Icon hat und ohne Terminalwissen startet.

Dieser Guide zeigt pragmatisch, wie du aus eigenen Java- oder Python-Projekten installierbare Anwendungen machst. Der Fokus liegt auf Windows und Linux, weil dort sehr unterschiedliche Erwartungen gelten: Windows-Nutzer erwarten meistens eine Setup.exe oder .msi; Linux-Nutzer erwarten eher ein .deb, .rpm, AppImage, Flatpak oder mindestens eine saubere .desktop-Datei.

Was ein Installer eigentlich macht

Ein Installer ist im Kern kein magisches Format. Er erledigt meistens diese Aufgaben:

  • Dateien an einen sinnvollen Ort kopieren.
  • Eine Startdatei oder einen Launcher anlegen.
  • Ein Icon registrieren.
  • Eine Desktop-Verknüpfung erstellen.
  • Einen Startmenü- oder App-Menü-Eintrag erstellen.
  • Optional einen Uninstaller erzeugen.
  • Optional Abhängigkeiten mitliefern, etwa Java Runtime, Python Runtime oder DLLs.

Der wichtigste Unterschied ist deshalb nicht "Installer oder kein Installer", sondern: Wie viel davon willst du selbst skripten und wie viel soll ein Packaging-Tool für dich erledigen?

ZielEinfacher WegProfessioneller Weg
Python-Skript als Windows-AppPyInstallerPyInstaller + Inno Setup
Python-GUI mit TkinterPyInstaller --windowedBriefcase oder PyInstaller + Installer
Java-App als Desktop-Appjava -jar + Shortcutjpackage
Linux-App sichtbar machen.desktop-Datei.deb, .rpm, AppImage oder Flatpak
Eigener Mini-InstallerPython-CLIPython + Tkinter-UI

Empfohlener Ablauf

  1. Baue zuerst eine lauffähige App in einem Ordner, zum Beispiel dist/MeineApp/.
  2. Lege ein Icon ab, unter Windows als .ico, unter Linux meist als .png.
  3. Teste den Start lokal ohne Entwicklungsumgebung.
  4. Erstelle daraus einen Installer oder ein Paket.
  5. Veröffentliche den Installer auf einer Webseite, GitHub Releases, einer Schul-/Uni-Seite oder einem internen Downloadbereich.

Gerade für kleine Projekte ist es sinnvoll, nicht bei jedem Nutzer eine komplizierte Build-Anleitung vorauszusetzen. Besser ist: Du baust lokal oder in CI einen fertigen Installer, stellst ihn auf eine Webseite und die Nutzer laden nur noch MeineApp-Setup.exe, MeineApp.deb oder MeineApp.AppImage herunter.

Installer von einer Webseite ziehen

Wenn du einen Installer verteilst, ist eine klare Downloadseite oft besser als ein Repository. Auf GitHub kann das ein Release sein; auf einer eigenen Webseite kann es ein einfacher Downloadbutton sein. Wichtig ist:

  • Die Datei sollte einen eindeutigen Namen haben: MeineApp-1.0.0-windows-x64-setup.exe.
  • Die Webseite sollte Version, Betriebssystem und Datum nennen.
  • Bei größeren Projekten ist ein SHA256-Hash sinnvoll.
  • Windows kann bei unbekannten .exe-Dateien warnen. Das ist normal, solange die Datei nicht signiert ist.
  • Lade fremde Installer nur von offiziellen Projektseiten oder Releases herunter, nicht von zufälligen Mirror-Seiten.
# SHA256 unter Windows prüfen
certutil -hashfile MeineApp-Setup.exe SHA256

# SHA256 unter Linux prüfen
sha256sum MeineApp.AppImage

Windows: Python-App als EXE bauen

Für Python ist PyInstaller der schnellste Einstieg. Es analysiert dein Skript, sammelt Python, Module und Ressourcen ein und erzeugt daraus einen Ordner oder eine einzelne .exe.

python -m venv .venv
.venv\Scripts\activate
pip install pyinstaller

# CLI-App
pyinstaller --onefile --name MeineApp main.py

# GUI-App ohne schwarzes Konsolenfenster
pyinstaller --onefile --windowed --name MeineApp --icon assets\app.ico main.py

Bei einer Tkinter-App ist --windowed wichtig, weil sonst neben dem GUI-Fenster ein Konsolenfenster geöffnet wird.

Dateien mitliefern

Wenn deine App Bilder, JSON-Dateien, Datenbanken oder Modelle braucht, musst du sie mitgeben. Unter Windows trennt PyInstaller Quelle und Ziel bei --add-data mit Semikolon:

pyinstaller --onefile --windowed ^
  --name Stundenplan ^
  --icon assets\stundenplan.ico ^
  --add-data "assets;assets" ^
  --add-data "data;data" ^
  main.py

Unter Linux wird bei --add-data ein Doppelpunkt verwendet:

pyinstaller --onefile --windowed \
  --name stundenplan \
  --icon assets/stundenplan.png \
  --add-data "assets:assets" \
  --add-data "data:data" \
  main.py

Nach dem Build liegt die App unter dist/. Für interne Tools reicht das manchmal schon. Wenn die App aber richtig installiert werden soll, kommt ein Installer-Tool dazu.

Windows: Setup.exe mit Inno Setup

Inno Setup ist für Windows ein sehr praktischer Weg, aus einem fertigen App-Ordner einen Installer zu bauen. Man installiert Inno Setup von der offiziellen Webseite, schreibt eine .iss-Datei und kompiliert daraus eine Setup.exe.

Angenommen, deine PyInstaller-App liegt hier:

dist/
  MeineApp/
    MeineApp.exe
    assets/
    data/
assets/
  app.ico

Dann kann ein einfacher Inno-Setup-Script so aussehen:

; installer.iss
[Setup]
AppId={{8A2B14A2-9A4B-4F5A-8B9A-0E1F2A3B4C5D}
AppName=MeineApp
AppVersion=1.0.0
DefaultDirName={autopf}\MeineApp
DefaultGroupName=MeineApp
OutputDir=installer
OutputBaseFilename=MeineApp-Setup
SetupIconFile=assets\app.ico
Compression=lzma
SolidCompression=yes

[Files]
Source: "dist\MeineApp\*"; DestDir: "{app}"; Flags: recursesubdirs createallsubdirs

[Icons]
Name: "{autoprograms}\MeineApp"; Filename: "{app}\MeineApp.exe"; WorkingDir: "{app}"; IconFilename: "{app}\MeineApp.exe"
Name: "{autodesktop}\MeineApp"; Filename: "{app}\MeineApp.exe"; WorkingDir: "{app}"; IconFilename: "{app}\MeineApp.exe"; Tasks: desktopicon

[Tasks]
Name: "desktopicon"; Description: "Desktop-Verknüpfung erstellen"; GroupDescription: "Optionale Verknüpfungen:"

[Run]
Filename: "{app}\MeineApp.exe"; Description: "MeineApp starten"; Flags: nowait postinstall skipifsilent

Damit passiert genau das, was man von einer normalen Windows-App erwartet: Installation unter Programme, Startmenü-Eintrag, optional Desktop-Verknüpfung, Icon und Startoption am Ende.

Inno Setup per CLI bauen

Du kannst den Installer auch ohne GUI bauen. Das ist praktisch für Skripte oder GitHub Actions:

"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss

Danach liegt die fertige MeineApp-Setup.exe im konfigurierten OutputDir.

Eigener Mini-Installer als Python-CLI

Für Schulprojekte, interne Tools oder Bastelprojekte kann ein eigener kleiner Installer reichen. Der kopiert die App in einen Benutzerordner und legt eine Startverknüpfung an. Das ist nicht so professionell wie Inno Setup, aber sehr gut zum Verstehen.

Windows-Variante

# install_windows.py
from pathlib import Path
import os
import shutil
import subprocess

APP_NAME = "MeineApp"
SOURCE_DIR = Path("dist/MeineApp").resolve()
INSTALL_DIR = Path(os.environ["LOCALAPPDATA"]) / APP_NAME
EXE_PATH = INSTALL_DIR / "MeineApp.exe"
ICON_PATH = EXE_PATH

def copy_app():
    if INSTALL_DIR.exists():
        shutil.rmtree(INSTALL_DIR)
    shutil.copytree(SOURCE_DIR, INSTALL_DIR)

def create_shortcut(name, target_dir):
    shortcut = target_dir / f"{name}.lnk"
    ps = f'''
    $shell = New-Object -ComObject WScript.Shell
    $shortcut = $shell.CreateShortcut("{shortcut}")
    $shortcut.TargetPath = "{EXE_PATH}"
    $shortcut.WorkingDirectory = "{INSTALL_DIR}"
    $shortcut.IconLocation = "{ICON_PATH}"
    $shortcut.Save()
    '''
    subprocess.run(["powershell", "-NoProfile", "-ExecutionPolicy", "Bypass", "-Command", ps], check=True)

def main():
    copy_app()
    desktop = Path(os.environ["USERPROFILE"]) / "Desktop"
    start_menu = Path(os.environ["APPDATA"]) / "Microsoft/Windows/Start Menu/Programs"
    create_shortcut(APP_NAME, desktop)
    create_shortcut(APP_NAME, start_menu)
    print(f"{APP_NAME} wurde installiert nach: {INSTALL_DIR}")

if __name__ == "__main__":
    main()

Dieser Installer braucht keine Adminrechte, weil er nach %LOCALAPPDATA% installiert. Wenn du nach C:\Program Files installieren willst, brauchst du Adminrechte und solltest eher Inno Setup verwenden.

Linux-Variante

# install_linux.py
from pathlib import Path
import os
import shutil
import stat

APP_NAME = "MeineApp"
SOURCE_DIR = Path("dist/MeineApp").resolve()
INSTALL_DIR = Path.home() / ".local/share" / APP_NAME
BIN_DIR = Path.home() / ".local/bin"
DESKTOP_DIR = Path.home() / ".local/share/applications"
ICON_PATH = INSTALL_DIR / "assets/app.png"
EXEC_PATH = INSTALL_DIR / "MeineApp"

def copy_app():
    if INSTALL_DIR.exists():
        shutil.rmtree(INSTALL_DIR)
    shutil.copytree(SOURCE_DIR, INSTALL_DIR)
    EXEC_PATH.chmod(EXEC_PATH.stat().st_mode | stat.S_IEXEC)

def create_launcher():
    BIN_DIR.mkdir(parents=True, exist_ok=True)
    launcher = BIN_DIR / "meineapp"
    launcher.write_text(f'#!/usr/bin/env bash\n"{EXEC_PATH}" "$@"\n', encoding="utf-8")
    launcher.chmod(0o755)

def create_desktop_file():
    DESKTOP_DIR.mkdir(parents=True, exist_ok=True)
    desktop_file = DESKTOP_DIR / "meineapp.desktop"
    desktop_file.write_text(f"""[Desktop Entry]
Type=Application
Name=MeineApp
Comment=Meine installierte App
Exec={EXEC_PATH}
Icon={ICON_PATH}
Terminal=false
Categories=Utility;
""", encoding="utf-8")

def main():
    copy_app()
    create_launcher()
    create_desktop_file()
    print("Installiert. Falls nötig: abmelden/anmelden oder App-Menü neu laden.")

if __name__ == "__main__":
    main()

Linux-Desktop-Umgebungen lesen .desktop-Dateien aus ~/.local/share/applications. Dadurch erscheint die App im Anwendungsmenü. Der Launcher unter ~/.local/bin sorgt dafür, dass man sie auch im Terminal starten kann.

Eigener Installer mit Tkinter-UI

Wenn der Installer nicht nur im Terminal laufen soll, kannst du eine kleine Tkinter-Oberfläche bauen. Das Prinzip bleibt gleich: Zielpfad auswählen, Dateien kopieren, Shortcut erstellen.

# installer_gui.py
from pathlib import Path
import os
import shutil
import subprocess
import tkinter as tk
from tkinter import filedialog, messagebox

APP_NAME = "MeineApp"
SOURCE_DIR = Path("dist/MeineApp").resolve()

def create_windows_shortcut(exe_path, install_dir, create_desktop):
    targets = []
    start_menu = Path(os.environ["APPDATA"]) / "Microsoft/Windows/Start Menu/Programs"
    targets.append(start_menu)
    if create_desktop:
        targets.append(Path(os.environ["USERPROFILE"]) / "Desktop")

    for target_dir in targets:
        shortcut = target_dir / f"{APP_NAME}.lnk"
        ps = f'''
        $shell = New-Object -ComObject WScript.Shell
        $shortcut = $shell.CreateShortcut("{shortcut}")
        $shortcut.TargetPath = "{exe_path}"
        $shortcut.WorkingDirectory = "{install_dir}"
        $shortcut.IconLocation = "{exe_path}"
        $shortcut.Save()
        '''
        subprocess.run(["powershell", "-NoProfile", "-ExecutionPolicy", "Bypass", "-Command", ps], check=True)

def install():
    target = Path(path_var.get()).expanduser()
    install_dir = target / APP_NAME
    try:
        if install_dir.exists():
            shutil.rmtree(install_dir)
        shutil.copytree(SOURCE_DIR, install_dir)
        exe_path = install_dir / "MeineApp.exe"
        create_windows_shortcut(exe_path, install_dir, desktop_var.get())
        messagebox.showinfo("Fertig", f"{APP_NAME} wurde installiert.")
    except Exception as exc:
        messagebox.showerror("Fehler", str(exc))

def choose_path():
    selected = filedialog.askdirectory(initialdir=path_var.get())
    if selected:
        path_var.set(selected)

root = tk.Tk()
root.title(f"{APP_NAME} Installer")
root.resizable(False, False)

default_dir = Path(os.environ["LOCALAPPDATA"])
path_var = tk.StringVar(value=str(default_dir))
desktop_var = tk.BooleanVar(value=True)

tk.Label(root, text=f"{APP_NAME} installieren").grid(row=0, column=0, columnspan=3, padx=16, pady=(16, 8), sticky="w")
tk.Entry(root, textvariable=path_var, width=48).grid(row=1, column=0, padx=(16, 6), pady=8)
tk.Button(root, text="Durchsuchen", command=choose_path).grid(row=1, column=1, padx=(0, 16), pady=8)
tk.Checkbutton(root, text="Desktop-Verknüpfung erstellen", variable=desktop_var).grid(row=2, column=0, columnspan=2, padx=16, pady=8, sticky="w")
tk.Button(root, text="Installieren", command=install).grid(row=3, column=0, columnspan=2, padx=16, pady=(8, 16), sticky="ew")

root.mainloop()

Das ist kein Ersatz für einen professionellen Windows-Installer, aber für kleine private Tools sehr brauchbar. Der große Vorteil: Du verstehst genau, was bei einer Installation passiert.

Java-Anwendungen als Installer

Für Java ist jpackage der sauberste Standardweg. Es ist Teil moderner JDKs und kann Java-Anwendungen in native Installer verpacken. Wichtig: Du baust Windows-Installer auf Windows und Linux-Pakete auf Linux. Cross-Builds sind nicht der Normalfall.

Vorbereitung

projekt/
  target/
    meineapp.jar
  libs/
    dependency-a.jar
    dependency-b.jar
  assets/
    app.ico
    app.png

Windows-Installer für Java

jpackage ^
  --type exe ^
  --name MeineJavaApp ^
  --input target ^
  --main-jar meineapp.jar ^
  --main-class de.beispiel.Main ^
  --icon assets\app.ico ^
  --win-menu ^
  --win-shortcut ^
  --app-version 1.0.0 ^
  --dest installer

--win-menu erzeugt einen Startmenü-Eintrag, --win-shortcut eine Desktop-Verknüpfung. Das Icon muss unter Windows typischerweise eine .ico-Datei sein.

Linux-Paket für Java

jpackage \
  --type deb \
  --name meine-java-app \
  --input target \
  --main-jar meineapp.jar \
  --main-class de.beispiel.Main \
  --icon assets/app.png \
  --linux-shortcut \
  --app-version 1.0.0 \
  --dest installer

Für Fedora/OpenSUSE wärdest du statt --type deb eher --type rpm verwenden. Für lokale Tests reicht auch --type app-image; dann bekommst du erst einmal einen App-Ordner ohne Paketinstallation.

Python-Anwendungen als richtige App

Für Python gibt es zwei typische Wege:

  • PyInstaller: Sehr praktisch für einzelne Tools, Tkinter-Apps, CLI-Skripte und Schul-/Uni-Projekte.
  • Briefcase: Baut eher native App-Projektstrukturen und Installer. Das ist interessant, wenn du eine Python-App langfristig plattformübergreifend pflegen willst.

Ein wichtiger Hinweis: Nicht jedes Packaging-Tool passt zu Tkinter gleich gut. Briefcase verwendet unter Windows die Python-Embeddable-Distribution; dort ist Tkinter nicht immer problemlos verfügbar. Für einfache Tkinter-Apps ist PyInstaller deshalb oft der direktere Weg.

CLI-Tool

# main.py
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("name")
args = parser.parse_args()
print(f"Hallo {args.name}")
pyinstaller --onefile --name hallo main.py

Tkinter-App

# main.py
import tkinter as tk

root = tk.Tk()
root.title("Meine Tkinter App")
tk.Label(root, text="Hallo Desktop!").pack(padx=30, pady=20)
root.mainloop()
pyinstaller --onefile --windowed --name MeineTkApp --icon assets\app.ico main.py

Danach kannst du die erzeugte .exe mit Inno Setup weiterverpacken.

Java und Python zusammen ausliefern

Manchmal besteht ein Projekt aus mehreren Teilen: zum Beispiel eine Java-GUI und ein Python-Skript für Datenanalyse, OCR, KI, Export oder Automatisierung. Dann hast du drei Möglichkeiten:

AnsatzWann sinnvoll?Nachteil
Python separat mit PyInstaller bauenPython-Teil ist ein HilfsprogrammMehrere Artefakte im Installer
Java startet Python-EXEJava ist Haupt-App, Python nur WorkerPfad- und Fehlerbehandlung nötig
Alles in eine Sprache portierenLangfristig wartbare AppMehr Entwicklungsaufwand

Beispielstruktur

dist/
  MeineSuite/
    MeineJavaApp.exe
    worker/
      analyse.exe
    assets/
      app.ico

Die Java-App kann den Python-Worker dann relativ zum eigenen Installationsordner starten:

// Java
Path appDir = Paths.get(System.getProperty("user.dir"));
Path worker = appDir.resolve("worker").resolve("analyse.exe");

ProcessBuilder pb = new ProcessBuilder(worker.toString(), "input.csv");
pb.directory(appDir.toFile());
Process process = pb.start();
int exitCode = process.waitFor();

Der Installer muss nur den gesamten Ordner kopieren und den Shortcut auf die Java-Hauptanwendung zeigen lassen.

Linux: App sichtbar machen

Unter Linux ist die App-Integration meist eine .desktop-Datei. Beispiel:

[Desktop Entry]
Type=Application
Name=MeineApp
Comment=Mein kleines Tool
Exec=/home/felix/.local/share/MeineApp/MeineApp
Icon=/home/felix/.local/share/MeineApp/assets/app.png
Terminal=false
Categories=Utility;Education;

Speicherort für den aktuellen Benutzer:

~/.local/share/applications/meineapp.desktop

Wichtig ist, dass Exec und Icon auf echte Dateien zeigen. Manche Desktop-Umgebungen brauchen danach ein Ab- und Anmelden oder einen Menü-Reload.

AppImage, Flatpak, deb, rpm

  • AppImage: Eine Datei, die viele Distributionen direkt ausführen können. Gut für einfache Downloads.
  • Flatpak: Gut für Distribution über Flathub oder kontrollierte Sandbox-Umgebung.
  • deb: Für Debian, Ubuntu, Linux Mint und verwandte Systeme.
  • rpm: Für Fedora, RHEL, openSUSE und verwandte Systeme.

Für Java kann jpackage direkt deb oder rpm erzeugen. Für Python ist PyInstaller plus AppImage oder ein eigenes .deb-Paket ein typischer Weg.

Praktische Usecases

Usecase 1: Schul-Tool mit Tkinter

Du hast ein kleines Tkinter-Programm, das Sitzpläne, Vokabeln oder Aufgaben verwaltet. Baue eine .exe mit PyInstaller und verpacke sie mit Inno Setup. Ergebnis: Lehrkräfte bekommen eine Setup-Datei, installieren das Tool und finden es im Startmenü.

Usecase 2: JavaFX-Lernapp

Du hast eine JavaFX-App geschrieben. Verwende jpackage, damit Java nicht separat installiert werden muss. Ergebnis: Windows-Installer oder Linux-Paket mit Icon und Menüeintrag.

Usecase 3: Python-Datenanalyse als Hilfsprogramm

Du hast eine Java-Oberfläche, aber die Datenanalyse ist in Python geschrieben. Baue das Python-Skript mit PyInstaller zu analyse.exe und liefere es im Unterordner worker/ mit. Die Java-App startet es bei Bedarf.

Usecase 4: Interne CLI für wiederkehrende Aufgaben

Du hast ein Python-CLI-Tool, das Dateien sortiert, PDFs verarbeitet oder Daten exportiert. Baue es mit PyInstaller als --onefile. Unter Windows kann Inno Setup den Installationspfad in eine feste App-Struktur bringen; unter Linux reicht oft ein Launcher in ~/.local/bin.

Prompt-Vorlagen für Coding Agents

Wenn du mit einem Coding Agent arbeitest, solltest du nicht nur schreiben: "Mach daraus einen Installer". Besser ist ein konkreter Auftrag: Der Agent soll erst das Projekt analysieren, dann eine installierbare Zielstruktur vorschlagen, danach Build- und Installer-Skripte erzeugen und am Ende prüfen, ob die Pfade, Icons und Startdateien wirklich stimmen.

Prompt: Bestehendes Projekt in Installer-Struktur umbauen

Du arbeitest in meinem bestehenden Projekt. Bitte analysiere zuerst die aktuelle Struktur und baue sie dann so um, dass daraus eine installierbare Desktop-App gebaut werden kann.

Zielplattformen:
- Windows: EXE/App-Ordner plus optional Inno-Setup-Installer
- Linux: ausführbarer App-Ordner plus .desktop-Datei, optional deb/rpm/AppImage

Bitte erledige konkret:
1. Erkenne die Startdatei der App.
2. Erkenne, ob es eine Python-, Java-, JavaFX-, Tkinter-, CLI- oder gemischte App ist.
3. Schlage eine klare Zielstruktur vor, zum Beispiel:
   app/
   assets/
   scripts/
   installer/
   dist/
   docs/
4. Verschiebe nur Dateien, wenn es sinnvoll und sicher ist. Erkläre jede Strukturänderung kurz.
5. Lege Build-Skripte an:
   - scripts/build-windows.bat
   - scripts/build-linux.sh
6. Lege Installer-Skripte an:
   - installer/install_windows.py oder installer.iss
   - installer/install_linux.py oder eine .desktop-Datei
7. Sorge dafür, dass Icon, Arbeitsverzeichnis, Startpfad und Ressourcen korrekt verwendet werden.
8. Füge eine kurze README-Sektion hinzu: Build, Installation, Deinstallation, typische Fehler.
9. Führe keine riskanten Löschaktionen aus. Wenn Dateien verschoben werden müssen, erkläre vorher den Plan.
10. Prüfe am Ende mit passenden Befehlen, ob die Build-Skripte syntaktisch plausibel sind und alle referenzierten Dateien existieren.

Prompt: Neues Projekt direkt mit installierbarer Struktur planen

Ich möchte ein neues Desktop-Projekt starten. Bitte erstelle mir eine saubere Projektstruktur, die von Anfang an für Packaging und Installer geeignet ist.

Rahmen:
- App-Name: <APP_NAME>
- Sprache/Stack: <Python mit Tkinter / Python CLI / Java / JavaFX / gemischt>
- Zielplattformen: Windows und Linux
- Icon-Dateien: assets/app.ico für Windows, assets/app.png für Linux
- Ressourcen: assets/ und data/

Bitte generiere:
1. Eine empfohlene Ordnerstruktur.
2. Eine minimale lauffähige Beispiel-App.
3. Ein Windows-Build-Skript als kopierbaren .bat-Code.
4. Ein Linux-Build-Skript als kopierbaren .sh-Code.
5. Einen einfachen Windows-Installer oder eine Inno-Setup-Vorlage.
6. Eine Linux-.desktop-Datei oder ein install_linux.py-Skript.
7. Eine README mit den Befehlen:
   - App lokal starten
   - Windows-Build erzeugen
   - Linux-Build erzeugen
   - Installer erzeugen
   - App installieren
8. Achte darauf, dass alle Pfade relativ zum Projekt funktionieren.

Interaktives Struktur-Skript für Windows

Dieses .bat-Skript fragt die wichtigsten Entscheidungen ab und erzeugt daraus eine projektnahe Grundstruktur. Es erzeugt noch keine fertige App, aber eine sehr gute Vorlage für den nächsten Agent-Schritt.

@echo off
setlocal EnableDelayedExpansion

echo === Desktop-App Projektstruktur Generator ===
set /p APP_NAME=App-Name [MeineApp]: 
if "%APP_NAME%"=="" set APP_NAME=MeineApp

set /p STACK=Stack python-cli, python-tkinter, java, javafx, mixed [python-tkinter]: 
if "%STACK%"=="" set STACK=python-tkinter

set /p MAIN_FILE=Startdatei [app\main.py oder app\Main.java]: 
if "%MAIN_FILE%"=="" set MAIN_FILE=app\main.py

set /p JAVA_MAIN=Java Main-Class, falls Java genutzt wird [de.beispiel.Main]: 
if "%JAVA_MAIN%"=="" set JAVA_MAIN=de.beispiel.Main

set /p WIN_ICON=Windows Iconpfad [.ico] [assets\app.ico]: 
if "%WIN_ICON%"=="" set WIN_ICON=assets\app.ico

set /p LINUX_ICON=Linux Iconpfad [.png] [assets/app.png]: 
if "%LINUX_ICON%"=="" set LINUX_ICON=assets/app.png

set /p WANT_INSTALLER=Installer-Vorlagen erzeugen? j/n [j]: 
if "%WANT_INSTALLER%"=="" set WANT_INSTALLER=j

mkdir "%APP_NAME%"
cd "%APP_NAME%"

mkdir app assets data scripts installer dist docs
type nul > assets\app.ico
type nul > assets\app.png
type nul > data\.gitkeep

(
echo # %APP_NAME%
echo.
echo Stack: %STACK%
echo Startdatei: %MAIN_FILE%
echo Java Main-Class: %JAVA_MAIN%
echo Windows-Icon: %WIN_ICON%
echo Linux-Icon: %LINUX_ICON%
echo.
echo ## Befehle
