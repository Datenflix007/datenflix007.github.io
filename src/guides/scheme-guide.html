Funktionale Programmierung Scheme

Scheme Guide für Dummies

Ein einfacher Einstieg in Scheme als funktionale Sprache mit Listen, Rekursion und Funktionen höherer Ordnung.

Scheme ist ein Dialekt von Lisp und gehört zur Familie der funktionalen Sprachen. Anders als bei Prolog steht nicht die logische Schlussfolgerung im Mittelpunkt, sondern die Auswertung von Funktionen und die kompositionale Verarbeitung von Daten.

Funktional

Programme bestehen aus Funktionen, die Daten transformieren.

Listenbasiert

Listen sind die zentrale Datenstruktur und werden oft rekursiv verarbeitet.

Komposition

Kleine Bausteine werden zu größeren Programmen zusammengesetzt.

1. Grundidee

In Scheme definiert man Funktionen und ruft sie mit Argumenten auf. Das Ergebnis entsteht durch Auswertung dieser Funktionen. Das ist sehr direkt und oft sehr kompakt.

(define (quadrat x)
  (* x x))

(quadrat 5)
25

2. Warum in Lisp alles als Liste funktioniert

Der zentrale Gedanke von Lisp und damit auch von Scheme ist: Programme sind Daten und Daten sind Programme. Das bedeutet, dass du Code nicht als etwas Besonderes neben den Daten ansiehst, sondern als eine Form von Liste, die ausgewertet wird. Eine einfache Liste wie (+ 2 3) ist zunächst einmal nur eine Liste aus Symbolen und Zahlen. Erst wenn der Interpreter diese Liste auswertet, wird daraus ein Funktionsaufruf.

Das ist der Grund, warum Lisp sehr elegant wirkt: Man arbeitet mit derselben Grundstruktur, mit derselben Listenform, ob man nun Zahlen, Wörter, Ausdrücke, Funktionen oder Programme modelliert.

42
'Hallo
(+ 2 3)
(list 1 2 3)

Die erste Zeile ist einfach eine Zahl. Die zweite Zeile ist ein Symbol oder ein Wort. Die dritte Zeile ist eine Liste, die als Funktionsaufruf interpretiert wird. Die vierte Zeile ist eine Liste, die als Datenstruktur verwendet wird.

Wichtiger Unterschied

Ein Ausdruck wie (+ 2 3) wird ausgeführt, weil er als Aufruf einer Funktion gelesen wird. Eine Liste wie '(1 2 3) wird dagegen als Datenliste behandelt, weil sie mit einem Apostroph markiert ist.

Quote und Datenlisten

(+ 2 3)
'(+ 2 3)

Die erste Zeile wird ausgewertet und ergibt 5. Die zweite Zeile ist eine Datenliste, die nur den Ausdruck selbst als Liste repräsentiert. Das ist ein sehr charakteristisches Lisp-Muster.

3. Prädikate und Bedingungen

In Scheme verwendet man oft if oder cond, um Entscheidungen zu treffen. Ein Prädikat ist dabei einfach ein Ausdruck, der wahr oder falsch ergibt.

(define (positiv? x)
  (> x 0))

(positiv? 5)
(positiv? -2)
#t
#f

Das ? am Ende eines Namens ist in Lisp-Style eine Konvention für Prädikate. Es signalisiert: „Diese Funktion liefert einen booleschen Wert.“

(define (groesser-als-null? x)
  (if (> x 0)
      #t
      #f))

4. Listen und Rekursion

Listen sind in Scheme sehr wichtig. Eine typische Lösung ist deshalb oft rekursiv: Basisfall plus rekursiver Fall.

(define (laenge lst)
  (if (null? lst)
      0
      (+ 1 (laenge (cdr lst)))))

(laenge '(1 2 3 4))
4

5. Funktionen höherer Ordnung

Scheme macht viel mit Funktionen höherer Ordnung wie map, filter und fold.

(define (verdoppeln lst)
  (map (lambda (x) (* x 2)) lst))

(verdoppeln '(1 2 3))
(2 4 6)

6. Konstruktoren und Selektoren

In Lisp arbeitet man oft mit Konstruktoren wie cons und Selektoren wie car und cdr. Das ist ein sehr wichtiger Teil der Zusammenfassung: Eine Liste ist ein Paar aus erstem Element und Restliste.

(define lst '(1 2 3))
(cons 0 lst)
(0 1 2 3)
(car lst)
(cdr lst)
1
(2 3)

car gibt das erste Element, cdr den Rest zurück. Diese beiden Operatoren sind in Lisp sehr fundamental.

7. Spezialformen und Auswertung

In Scheme gibt es Ausdrücke, die nicht einfach nur Funktionen sind. Dazu gehören if, define, lambda und quote. Diese werden nicht wie normale Funktionen behandelt, sondern haben eine eigene Auswertungsregel.

(define x 10)
(if (> x 5) 'groesser 'kleiner)
groesser

Das ist wichtig, weil Scheme eine sehr kleine und zugleich sehr mächtige Sprache ist: Die Syntax ist simpel, aber die Semantik wird durch die Auswertungsregeln bestimmt.

8. Vergleich: Prolog vs. Scheme

AspektPrologScheme
Zentraler StilFakten, Regeln, AnfragenFunktionen, Daten, Auswertung
Denkenlogisch und relationalfunktional und kompositorisch
WiederholungRekursion und BacktrackingRekursion und Funktionen höherer Ordnung
Typische StärkeRegelbasierte Suchekompakte Datenverarbeitung

Merksatz

Prolog fragt: „Was folgt logisch aus diesen Fakten?“ Scheme fragt eher: „Wie kann ich diese Daten mit Funktionen elegant transformieren?“

9. So wird aus Scheme wirklich ein praktisches Programm

Der beste Weg, Scheme zu lernen, ist nicht nur „ein paar Ausdrücke lesen“, sondern ein kleines Projekt von Anfang bis Ende durchzudenken. Die Idee ist dabei immer dieselbe: Du stellst dir zuerst ein Problem vor, dann zerlegst du es in kleine Schritte, und dann baust du die Lösung mit kleinen Funktionen auf.

Ein guter praktischer Ablauf sieht so aus:

  1. Problem beschreiben: Was soll das Programm tun?
  2. Eingaben und Ausgaben festlegen: Was bekommt die Funktion? Was gibt sie zurück?
  3. Kleine Funktionen schreiben: Nicht sofort alles auf einmal, sondern erst einen kleinen Baustein.
  4. Mit Beispielen testen: Was passiert bei 5, bei 0, bei einer leeren Liste?
  5. Schrittweise verallgemeinern: Aus einem Spezialfall wird eine allgemeine Funktion.

Beispielprojekt: Notenbewertung

Stell dir vor, du willst ein Programm schreiben, das aus einer Note eine Bewertung macht. Das ist ein klassisches Beispiel für eine kleine Regelmaschine.

(define (bewerte-note note)
  (cond
    [(>= note 90) "sehr gut"]
    [(>= note 80) "gut"]
    [(>= note 70) "befriedigend"]
    [else "nicht bestanden"]))

(bewerte-note 85)

Was passiert hier genau? Die Funktion bewerte-note bekommt eine Zahl. Dann schaut cond nacheinander, welche Bedingung zuerst zutrifft. Wenn die Note größer oder gleich 90 ist, gibt es „sehr gut“ zurück. Wenn nicht, wird die nächste Prüfung gemacht. Wenn keine Bedingung passt, kommt der else-Fall.

Beispielprojekt: Listen filtern

Ein zweites typisches Projekt ist das Verarbeiten von Listen. Das ist in Scheme besonders wichtig, weil Listen eine zentrale Datenstruktur sind.

(define (filter-gerade lst)
  (cond
    [(null? lst) '()]
    [(even? (car lst)) (cons (car lst) (filter-gerade (cdr lst)))]
    [else (filter-gerade (cdr lst))]))

(filter-gerade '(1 2 3 4 5 6))

Die Funktion schaut sich das erste Element an. Ist es gerade, wird es behalten. Ist es ungerade, wird es übersprungen. Danach wird der Rest der Liste weiterverarbeitet. Das ist ein typischer rekursiver Aufbau: Basisfall plus kleiner Schritt.

Beispielprojekt: Einfacher Wort- oder Textanalysator

Mit Scheme kannst du auch Text verarbeiten. Ein sehr kleines Beispiel wäre, zu prüfen, ob ein Wort mit einem bestimmten Anfangsbuchstaben beginnt.

(define (startet-mit-a? wort)
  (equal? (substring wort 0 1) "a"))

(startet-mit-a? "apfel")
(startet-mit-a? "banane")

Auch hier sieht man das Prinzip: Ein kleiner Input, eine klare Frage, ein klares Ergebnis. Genau solche kleinen Bausteine sind der Einstieg in größere Programme.

Diese drei Beispiele zeigen schon sehr gut, warum Scheme so attraktiv ist: Es ist kompakt, klar und sehr gut geeignet für die Verarbeitung von Regeln, Listen und kleinen Transformationen.

10. DrRacket installieren und sinnvoll benutzen

Wenn du Scheme wirklich ausprobieren willst, ist DrRacket der einfachste Einstieg. Es ist eine Entwicklungsumgebung für Racket und damit auch für Scheme-ähnliche Programmierung. Sie ist besonders angenehm, weil du sofort sehen kannst, was dein Code macht.

  1. Öffne deinen Browser und gehe auf racket-lang.org.
  2. Lade den Installer für dein Betriebssystem herunter. Auf Windows ist das meist eine .exe-Datei.
  3. Installiere Racket mit den Standardoptionen. Dabei wird auch DrRacket mitinstalliert.
  4. Öffne DrRacket. Dort siehst du meist ein Editorfenster und ein Interaktionsfenster.
  5. Wähle eine Sprache wie Racket oder Beginning Student. Für den Anfang ist Beginning Student oft besonders gut, weil es dich nicht sofort mit allen Spezialfällen überfordert.
  6. Schreibe deinen ersten Code und drücke Run.
#lang racket
(+ 2 3)

Wenn du das ausführst, sollte die Ausgabe 5 erscheinen. Das ist dein erster echter Lauf eines Scheme-ähnlichen Programms.

Wie du in DrRacket typischerweise arbeitest

1. Neues File anlegen
2. Code schreiben
3. Auf Run klicken
4. Ergebnisse im Interaktionsfenster ansehen
5. Bei Fehlern den Fehlertext lesen und den Code anpassen

Das Wichtigste ist: Nicht sofort alles perfekt machen wollen. Ein kleiner Fehler ist kein Problem. Gerade in Scheme lernt man oft sehr schnell dadurch, dass man den Fehlertext liest und den nächsten Versuch macht.

Ein erstes, bewusst einfaches Beispiel

#lang racket
(define (quadrat x)
  (* x x))

(quadrat 6)

Hier definierst du zuerst eine Funktion quadrat. Sie bekommt einen Wert x und gibt dessen Quadrat zurück. Danach rufst du die Funktion mit 6 auf. Das Ergebnis ist 36.

11. Vom kleinen Experiment zur echten Anwendung

Ein sehr praktischer Weg ist: Scheme für die eigentliche Logik zu verwenden und Python als Brücke zu einer Weboberfläche oder einer Cloud-Umgebung zu nutzen. Das ist ein sehr guter Lernweg, weil du dabei nicht nur die funktionale Denkweise lernst, sondern auch einen realistischen Aufbau siehst.

Die grobe Architektur ist:

Browser-UI → Python-API → Scheme-Logik → Ergebnis

Das heißt: Der Nutzer sieht nur die Oberfläche, aber die eigentliche Entscheidung oder Verarbeitung passiert in der Logik-Schicht. Dadurch bleibt dein Programm sauber getrennt.

Schritt 1: Die Logik in Scheme

#lang racket
(provide berechne-bonus)

(define (berechne-bonus alter)
  (cond
    [(< alter 18) 0]
    [(< alter 65) 50]
    [else 100]))

Diese Funktion ist die eigentliche Logik. Sie sagt: Wer unter 18 ist, bekommt keinen Bonus; wer zwischen 18 und 64 ist, bekommt 50; sonst bekommt 100.

Schritt 2: Python ruft die Scheme-Logik auf

import subprocess


def berechne_bonus(alter: int) -> int:
    command = ["racket", "-e", f"(require \"logic.rkt\") (display (berechne-bonus {alter}))"]
    result = subprocess.run(command, capture_output=True, text=True, check=True)
    return int(result.stdout.strip())

print(berechne_bonus(25))

Hier wird der Scheme-Code nicht direkt in Python neu geschrieben, sondern von Python aus aufgerufen. Das ist besonders praktisch, wenn du eine bestehende Regel- oder Entscheidungslogik getrennt pflegen möchtest.

Schritt 3: Ein kleines Python-Backend mit Flask

from flask import Flask, request, jsonify
import subprocess

app = Flask(__name__)

@app.post("/bonus")
def bonus():
    data = request.get_json()
    alter = data["alter"]
    command = ["racket", "-e", f"(require \"logic.rkt\") (display (berechne-bonus {alter}))"]
    result = subprocess.run(command, capture_output=True, text=True, check=True)
    return jsonify({"bonus": int(result.stdout.strip())})

if __name__ == "__main__":
    app.run(debug=True)

Dieses Backend macht aus deiner kleinen Logik einen echten Dienst, den man über eine URL ansprechen kann. Das ist der Schritt vom „einfachen Skript“ zur „echten Anwendung“.

Schritt 4: Eine einfache UI darüber

<input id="alter" type="number" value="25" />
<button onclick="senden()">Bonus berechnen</button>
<div id="ergebnis"></div>

<script>
async function senden() {
  const alter = Number(document.getElementById("alter").value);
  const response = await fetch("/bonus", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ alter })
  });
  const data = await response.json();
  document.getElementById("ergebnis").textContent = "Bonus: " + data.bonus;
}
</script>

Damit hast du mit wenig Aufwand eine echte kleine Weboberfläche. Der Nutzer tippt eine Zahl ein, die Website schickt sie an das Backend, das Backend ruft die Scheme-Logik auf, und die Antwort wird angezeigt.

12. Was du in der Cloud machen kannst

Wenn du das Ganze online stellen willst, gibt es verschiedene Möglichkeiten. Der wichtigste Gedanke ist: Die Logik bleibt getrennt, und die Oberfläche kann ganz einfach veröffentlicht werden.

  • Python-Backend auf Render, Fly.io oder Railway: Dort kannst du dein Flask-Backend deployen.
  • Frontend auf GitHub Pages oder Cloudflare Pages: Dort kannst du die HTML-/JS-Oberfläche kostenlos hosten.
  • Serverlose Funktionen: Wenn du nur einen kleinen Dienst brauchst, können Cloud-Funktionen sehr elegant sein.

Ein typischer Aufbau wäre also:

Browser-UI → Python-API → Scheme-Logik → Ergebnis

Das ist eine sehr gute Architektur für Lernprojekte, weil du dabei die Trennung von Darstellung, Verarbeitung und Logik direkt erleben kannst.

(define (summe lst)
